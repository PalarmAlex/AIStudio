НазваниеЗеркальные автоматизмы Эхо и Сдвиг. Добавление новых вариантов триггеров и ответов, склейка автоматизмов.
ОписаниеТестируется формирование зеркальных автоматизмов (эхо-автоматизм и сдвиг-автоматизм) в процессе диалога «Оператор — Агент» на 3 стадии развития агента.

═══════════════════════════════════════════════════════════════
I. Теоретические основания
═══════════════════════════════════════════════════════════════

Эхо-автоматизм:
Непосредственное воспроизведение предъявленного стимула в качестве ответного действия. Аналог эхолалии на ранних стадиях речевого развития у человека и имитационного научения у животных. Представляет собой элементарную форму подражания: стимул S копируется в ответ R, образуя пару S → S.

Сдвиг-автоматизм:
Формирование последовательной ассоциации между смежными стимулами в диалоговой цепочке. Предыдущий стимул (S_n−1) становится триггером, текущий стимул (S_n) — действием, образуя пару S_n−1 → S_n. Аналог последовательного обусловливания и формирования поведенческих цепочек в оперантных экспериментах (chaining).

Уровни убеждённости (Belief):
- Belief 2 — штатный (активный) автоматизм, привязанный к ветке дерева активации.
- Belief 0 — запасной (вытесненный) автоматизм, сохранённый, но не активируемый при стандартном обходе.

═══════════════════════════════════════════════════════════════
II. Механизм формирования
═══════════════════════════════════════════════════════════════

При предъявлении каждого нового вербального стимула S_n агент выполняет до двух операций:

1. Эхо-автоматизм: создаётся пара S_n → S_n (Belief 2), агент запускает её выполнение. Активируется период ожидания ответа оператора.

2. Сдвиг-автоматизм (при наличии активного периода ожидания от предыдущего шага): создаётся пара S_n−1 → S_n (Belief 2). При этом эхо-автоматизм предыдущего шага, привязанный к той же ветке, вытесняется в запасные (Belief 2 → Belief 0).

При повторном предъявлении стимула, для которого уже существуют автоматизмы, активируется штатный (Belief 2) без создания нового. Разрыв цепочки производится принудительным сбросом периода ожидания.

Формально, для диалоговой последовательности S₁, S₂, ..., S_n система порождает:
- Эхо-автоматизмы: {S_i → S_i}, i = 1..n (Belief 0 для вытесненных, Belief 2 для последнего)
- Сдвиг-автоматизмы: {S_i → S_i+1}, i = 1..n−1 (Belief 2)
═══════════════════════════════════════════════════════════════
III. Склейка цепочек
═══════════════════════════════════════════════════════════════

При завершении первой диалоговой цепочки (сброс ожидания) и начале второй происходит доращивание: конечный автоматизм первой цепочки дополняется недостающим сдвиг-автоматизмом, связывающим его с началом второй цепочки.

IV. Протокол тестирования
═══════════════════════════════════════════════════════════════

Цепочка 1 — формирование:

Шаг 1. Оператор: «привет». Агент учит эхо «привет → привет» (штатное, если на «привет» ещё ничего не было). Сдвига нет. Ждём оценку.
Шаг 2. Оператор: «хай». Сначала сдвиг «привет → хай» (штатный на «привет»), эхо «привет → привет» перестаёт быть штатным. Затем эхо «хай → хай» без штатного приоритета.
Шаг 3. Оператор: «как дела». Сначала сдвиг «хай → как дела» (штатный на «хай»). На «хай» до этого не было штатного эхо, вытеснять нечего. Затем эхо «как дела → как дела» без штатного приоритета.
Шаг 4. Оператор: «все ОК». Сначала сдвиг «как дела → все ОК» (штатный на «как дела»). Затем эхо «все ОК → все ОК» без штатного приоритета.
Шаг 5. Принудительный сброс времени ожидания. Цепочка 1 разорвана.

Цепочка 2 — активация существующих и доращивание:

Шаг 6. Оператор: «привет». Срабатывает выученный сдвиг «привет → хай». Якорь для следующего сдвига — «хай» (то, что скажет агент), а не «привет».
Шаг 7. Оператор: «как дела». Срабатывает «как дела → все ОК». Следующий якорь — «все ОК».
Шаг 8. Оператор: «это хорошо». Сначала сдвиг «все ОК → это хорошо» (штатный), затем эхо «это хорошо → это хорошо» без штатного приоритета.
Шаг 9. Принудительный сброс времени ожидания. Цепочка 2 разорвана.

Цепочка 3 — новый вариант ответа и триггера:

Шаг 10. Оператор: «приветствую». Агент учит эхо «приветствую → приветствую».
Шаг 11. Оператор: «хай». Срабатывает «хай → как дела». Создается сдвиг «приветствую → хай» (штатный). Следующий якорь — «как дела».
Шаг 12. Оператор: «все отлично». Сначала сдвиг «как дела → все отлично» (штатный), затем эхо «все отлично → все отлично» без штатного приоритета.
Шаг 13. Принудительный сброс времени ожидания. Цепочка 3 разорвана.

В результате на стимул "как дела" добавляется новый вариант ответа «как дела → все отлично», который теперь будет штатным. А остальные «как дела → как дела», «как дела → все ОК» переходят в разряд "запасных". Кроме того для ответа "хай" создается новый вербальный триггер «приветствую → хай». Таким образом в процессе диалогов образуются связи в автоматизмах между стимулами и действиями в отношении "многие ко многим": на один стимул может быть несколько ответов, один ответ может быть активирован несколькими стимулами.

Цепочка 4 — склейка существующих автоматизмов:

Шаг 14. Оператор: «привет, как дела». Агент разделяет строку на две части по запятой «привет" и "как дела», находит по ним штатные автоматизмы «привет → хай», «как дела → все отлично», и создает новый штатный автоматизм:
«привет, как дела → хай, все отлично».
